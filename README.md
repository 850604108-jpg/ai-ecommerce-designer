# AI Ecommerce Designer

AI Ecommerce Designer is a production-ready Next.js app for ecommerce creative workflows. It lets users upload a product photo, recognize product details with OpenAI, generate marketplace-specific prompts, create AI product images, manage generated image history, and receive credits through daily check-in.

## Stack

- Next.js 15 App Router
- React 19
- TypeScript
- Supabase Auth, Postgres, and Storage
- OpenAI Responses and Images APIs
- Vercel deployment target

## Features

- User registration, login, password reset, and session-aware navigation.
- Product image upload to Supabase Storage.
- AI product recognition from uploaded images.
- Prompt generation for Taobao, Tmall, Pinduoduo, JD, Douyin, Kuaishou, and WeChat Store.
- AI generation for main images, lifestyle images, infographics, and detail page modules.
- Credit ledger with daily check-in grants, spend records, and refunds.
- Dashboard with generated image history, search, pagination, copy, regenerate, and soft delete.
- Admin dashboard for users, projects, payments, credits, and templates.
- Template marketplace and admin template management.
- Production health check endpoint.
- SEO metadata, sitemap, robots, web manifest, and route-level loading skeletons.
- Optimized generated-image delivery with WebP output, Next.js image optimization, and long-lived cache headers.
- Unified API error responses with `error`, `code`, and `request_id`.

## Getting Started

Install dependencies:

```bash
npm install
```

Create local environment variables:

```bash
cp .env.example .env.local
```

Fill in Supabase, OpenAI, and app URL values in `.env.local`.

Run the development server:

```bash
npm run dev
```

Open `http://localhost:3000`.

## Production Optimizations

### SEO

- Global metadata lives in `src/app/layout.tsx`.
- Page-specific metadata is defined for the home page, generator, and dashboard.
- `src/app/sitemap.ts`, `src/app/robots.ts`, and `src/app/manifest.ts` are generated by Next.js.
- Authenticated/private areas such as dashboard, account, admin, and API routes are excluded from indexing.

### API Errors

API routes use `src/lib/api-response.ts` for consistent JSON responses. Error responses keep a backwards-compatible `error` message and also include:

```json
{
  "error": "Unauthorized.",
  "code": "UNAUTHORIZED",
  "request_id": "uuid"
}
```

Common codes include `BAD_REQUEST`, `UNAUTHORIZED`, `FORBIDDEN`, `PAYMENT_REQUIRED`, and `INTERNAL_ERROR`.

### Loading Skeletons

Next.js route loading states are provided by:

- `src/app/loading.tsx`
- `src/app/generate/loading.tsx`
- `src/app/dashboard/loading.tsx`
- `src/components/ui/skeleton.tsx`

### Images

- Generated images default to WebP in `src/lib/image-generation/jobs.ts`.
- Supabase Storage uploads use long cache control.
- `next/image` optimizes remote Supabase images through the remote pattern in `next.config.ts`.
- Next.js image output is configured for AVIF/WebP where supported.

## Deployment

Deploy to Vercel with the default Next.js settings. Full production deployment steps are in [docs/deployment.md](./docs/deployment.md).

Before deploying:

```bash
npm run lint
npm run build
npm run typecheck
npm run check:deployment
```

After deployment, check:

```bash
curl -H "Authorization: Bearer $HEALTHCHECK_TOKEN" https://your-domain.com/api/health/deployment
```

The endpoint returns safe pass/fail deployment checks and verifies the database connection when server credentials are configured.
